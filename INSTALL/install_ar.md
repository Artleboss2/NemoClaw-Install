# كيفية تثبيت NVIDIA NemoClaw: دليل خطوة بخطوة

NemoClaw عبارة عن حزمة برمجيات مفتوحة المصدر من NVIDIA مصممة لتشغيل وكلاء الذكاء الاصطناعي OpenClaw بأمان من خلال إضافة ضوابط للخصوصية، وبيئة تشغيل معزولة (sandboxing)، وحواجز أمنية.

اتبع هذه الخطوات السهلة لتشغيل وكيل الذكاء الاصطناعي الآمن الخاص بك.

## المتطلبات الأساسية

قبل البدء، تأكد من أن نظامك يلبي المتطلبات التالية:

* **نظام التشغيل:** لينكس (يُنصح باستخدام Ubuntu 22.04 أو أحدث). نظامي macOS و Windows (عبر WSL2) مدعومان ولكن بشكل تجريبي.
* **العتاد (Hardware):** ذاكرة وصول عشوائي (RAM) بسعة 8 جيجابايت على الأقل (يُنصح بشدة بـ 16 جيجابايت) وحوالي 40 جيجابايت من مساحة القرص الفارغة.
* **مفتاح واجهة برمجة التطبيقات (API Key):** مفتاح واجهة برمجة تطبيقات NVIDIA (يمكنك الحصول على واحد مجانًا من `build.nvidia.com`).

## الخطوة 1: تجهيز نظامك

يتطلب NemoClaw تثبيت Docker و Node.js (الإصدار 20 أو 22) ليعمل بشكل صحيح.

### 1. التحقق من Docker:

تأكد من تثبيت Docker وأن لديك الصلاحيات المناسبة لتشغيله دون استخدام `sudo` في كل مرة:

```bash
docker ps
```

(إذا ظهر لك خطأ برفض الصلاحية "permission denied"، قم بتشغيل `sudo usermod -aG docker $USER`، ثم قم بتسجيل الخروج وتسجيل الدخول مرة أخرى).

### 2. تثبيت Node.js:

قم بتشغيل الأوامر التالية لتثبيت Node.js (الإصدار 22):

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

تحقق من نجاح التثبيت عن طريق تشغيل `node --version`.

## الخطوة 2: تثبيت NemoClaw

توفر NVIDIA برنامجًا نصيًا (script) بسيطًا للتثبيت يتولى المهام المعقدة.

قم بتشغيل الأمر التالي في موجه الأوامر (Terminal):

```bash
curl -fsSL https://nvidia.com/nemoclaw.sh | bash
```

ملاحظة: بناءً على إعدادات نظامك، قد تحتاج إلى تشغيله باستخدام `sudo bash` إذا واجهت أخطاء تتعلق بالصلاحيات.

## الخطوة 3: إعداد الوكيل الخاص بك (Onboard Your Agent)

بمجرد اكتمال التثبيت، ستحتاج إلى تكوين البيئة المعزولة (sandbox) وإعدادات الاستدلال (inference).

قم بتشغيل معالج الإعداد:

```bash
nemoclaw onboard
```

أثناء هذه العملية التفاعلية:

* قم بتسمية المساعد الخاص بك (على سبيل المثال، `my-assistant`).
* أدخل مفتاح NVIDIA API الخاص بك عند المطالبة بذلك.
* اقبل الإعدادات الافتراضية للسياسات الأمنية وإعداد البيئة المعزولة.

ملاحظة: سيقوم البرنامج النصي بتنزيل صورة البيئة المعزولة (بحجم 2.4 جيجابايت تقريبًا). قد يستغرق هذا بضع دقائق.

## الخطوة 4: الاتصال والدردشة!

بيئتك الآمنة جاهزة الآن. حان الوقت للاتصال بوكيلك المعزول.

### 1. الاتصال بالبيئة المعزولة:

استبدل `my-assistant` بالاسم الذي اخترته في الخطوة 3.

```bash
nemoclaw my-assistant connect
```

### 2. بدء واجهة الدردشة:

بمجرد دخولك إلى البيئة المعزولة، قم بتشغيل واجهة المستخدم النصية (TUI) الخاصة بـ OpenClaw:

```bash
openclaw tui
```

تهانينا! يمكنك الآن كتابة الرسائل بأمان والتفاعل مع وكيل الذكاء الاصطناعي المعزول الخاص بك.

## نصائح استكشاف الأخطاء وإصلاحها

* **أخطاء نفاد الذاكرة (OOM):** إذا تعطل التثبيت على جهاز يحتوي على ذاكرة وصول عشوائي (RAM) بسعة 8 جيجابايت، فحاول إضافة مساحة تبديل (swap space) إلى نظامك.
* **مستخدمو WSL2:** إذا كنت تستخدم النظام الفرعي لويندوز لنظام لينكس (WSL2) وواجهت مشكلات في تمرير وحدة المعالجة الرسومية (GPU passthrough) أثناء تشغيل أمر `nemoclaw onboard`، فقد تحتاج إلى بدء بوابة OpenShell يدويًا بدون استخدام علامة `--gpu`.
